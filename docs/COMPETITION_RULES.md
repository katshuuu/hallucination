# Правила соревнования / жюри: время, железо, данные, метрики

Краткая фиксация требований для **production‑готового** решения и проверки жюри.

## Время детекции

- На **один ответ** действует ограничение по времени обработки: **500 ms** (по **учитываемому** времени, см. ниже).
- В **основном пайплайне** необходимо **замерять** время работы детектора галлюцинаций. В репозитории это сделано так:
  - HTTP **`POST /predict`**: поля `wall_time_ms`, `billable_time_ms`, `jury_max_billable_ms` (см. `configs/default.yaml` → `timing`);
  - CLI: `python -m hallucination_detector.cli benchmark`, `scripts/measure_performance.py` — выводят wall и **billable** рядом с лимитом.

### Что входит в «учитываемое» время (billable)

- **Forward** любых моделей в общем случае **учитывается**, если не выполняется исключение ниже.
- **Исключение для одной модели:** один forward модели **`ai-sage/GigaChat3-10B-A1.8B-bf16`** на последовательности токенов **prompt+response** (например, для снятия активаций со скрытых слоёв, см. `baseline.ipynb`) **можно не включать** в учитываемое время. Для этого в конфиге задаётся **`timing.gigachat_forward_excluded_ms`** — измеренная длительность этого одного forward (мс), которая **вычитается** из wall time при расчёте `billable_time_ms`.
- **Градиенты / backward:** если метод использует **анализ градиентов**, время **backward** **всегда учитывается** в полном объёме, **независимо от модели**; вычитание forward GigaChat **не применяется** (в конфиге выставьте **`timing.uses_gradient_analysis: true`** — тогда `billable_time_ms` = `wall_time_ms`).

Текущий **табличный** бейзлайн (sentence-transformers MiniLM + LightGBM) **не** использует GigaChat: по умолчанию `gigachat_forward_excluded_ms: 0`, весь замер — учитываемый.

## Железо финальной проверки

- Оценка проводится на **1× GPU NVIDIA A100 80GB**.
- Локальные замеры на CPU/другой GPU — ориентир; итоговое соблюдение **500 ms billable** нужно подтвердить на эталонной конфигурации.

## Метрики качества и равенство решений

- Основная метрика: **PR-AUC**.
- При **одинаковых** метриках лучшее решение определяется по **скорости** алгоритма (учитываемое время детекции).

## Данные и утечки

- **Сбор данных** под обучение — часть задачи; **источники не ограничены** (кроме запрета на внешние API в архитектуре инференса).
- Обучаться на **preview / публичном бенчмарке** **нельзя** — он для отладки и оценки своих идей; в проекте контроль пересечения с `knowledge_bench_public.csv` при обучении: `scripts/check_no_public_benchmark_overlap.py`, `training.allow_public_overlap`.

## Инференс

- **Встраивание внешних API** (облачные LLM для факт-чека и т.п.) в **архитектуру инференса** **не допускается**.
